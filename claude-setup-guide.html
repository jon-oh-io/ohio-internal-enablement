Claude Working
Directory Setup

A persistent memory system that lets Claude know who you are, how you work, and what you're working on — so you never re-explain yourself.

20–30 minutes
6 steps
No coding required

What you're building

📄

CLAUDE.md

A one-line entry point at the root of your working folder that tells Claude where to find your instructions.

📁

claude-context/

A folder of Markdown files Claude reads as standing instructions every session.

💼

portcos/

Per-client or per-project subfolders for source material and generated outputs.

🗂

internal-enablement/

Team-facing deliverables — guides, templates, and playbooks not tied to a specific client.

Step-by-step guide

1

Create Your Working Folder

Open Cowork and select (or create) a dedicated folder on your computer. Good names: Claude CoWork or [Your Name] Claude. This becomes your persistent workspace.

Create the following folder structure inside it. Leave the files empty for now — you'll populate them in later steps.

Claude CoWork/ ├── CLAUDE.md ← entry point ├── claude-context/ │ ├── INDEX.md │ └── operating-guide.md ├── portcos/ ← or clients/, projects/ └── internal-enablement/
✓

Create the folders now. The files will be generated for you in Step 3.

2

Create CLAUDE.md

This single file at the root of your working folder is the only instruction Claude reads automatically. Its entire job is to point Claude at your context folder.

Create CLAUDE.md with this content — updating the path to match your own folder location:

CLAUDE.md 
At the start of any non-trivial task, read /path/to/your/Claude CoWork/claude-context/INDEX.md and follow its guidance

How to find your path:

  • Mac: Right-click the claude-context folder → Get Info → copy the path under "Where"
  • Windows: Shift + right-click the folder → Copy as path
Example (Mac, user named Alex) 
At the start of any non-trivial task, read /Users/alex/Documents/Claude CoWork/claude-context/INDEX.md and follow its guidance
3

Run the Setup Interview

This is the most important step. Open a new Cowork session with your folder connected, then paste the prompt below. Claude will interview you one question at a time, then draft two files for your review:

  • operating-guide.md — how you want Claude to communicate, work, and produce output.
  • INDEX.md — a map of your context folder with instructions for how Claude uses it.
Paste this prompt into Cowork 
I'm setting up a Claude working directory. I want you to interview me to build two files:

1. operating-guide.md — how I want you to communicate, work, and produce output.
2. INDEX.md — a map of my claude-context folder with instructions for how you should use it.

Interview me with one question at a time. Cover these areas:
- My role and what I actually do day-to-day
- Who I work with and for (internal team, clients, stakeholders)
- The top 3–5 tasks I'll ask you to help with most often
- How I want you to communicate (tone, length, format, level of detail)
- How I want you to handle ambiguity — ask me or make a call?
- How I want you to handle disagreement — push back once, always defer, or something else?
- Any hard rules: things you should never do, never assume, never fabricate
- Any recurring workflows I have that you should recognize and structure automatically
- Confidentiality or data hygiene rules (e.g., test data to ignore, client names to keep separate)
- File output preferences (when to create files vs. respond in chat, preferred formats)

After the interview, draft both files and show them to me for review before saving anything. Once I approve, save them to my claude-context/ folder.
⚠

Tip: If Claude asks about something you haven't figured out yet, say so. An honest partial guide is far more useful than a fabricated complete one. The guide sharpens over time.

4

Review and Save the Files

After the interview, Claude will show you drafts of operating-guide.md and INDEX.md. Check them carefully before confirming:

  • Does the communication style match how you actually want to work?
  • Are the hard rules correct and complete?
  • Did Claude make any assumptions that are wrong?

Request any changes in the conversation, then confirm. Claude will save both files to claude-context/.

📌

The INDEX.md file is your map. Every time you add a new file to claude-context/, ask Claude to update the Contents table so the index stays accurate.

5

Test the Setup

Start a fresh Cowork session and give Claude a real task — something typical for your role. Verify that it:

  • Reads your operating guide before starting work
  • Matches your preferred tone and format without being told
  • Applies your hard rules automatically

If something feels off, go back to operating-guide.md and edit it directly, or ask Claude to update it based on what you observed.

6

Add Client or Project Context Files

Once your operating guide is in place, add a context file for each active client, project, or account. These are conditional — Claude only reads them when that subject is active.

File name When to create it
portco-<name>.md Per client — voice, buyers, terminology, do-not-use lists
template-<artifact>.md Reusable output templates (ROI models, discovery decks, one-pagers)
playbook-<topic>.md Repeatable workflows you run regularly (pre-call prep, post-call summaries)
glossary.md Shared vocabulary, acronyms, and internal terms

To generate a client context file, paste this prompt:

Client context prompt 
I want to create a context file for [client/project name]. Interview me to capture:
- What they do and who their buyers are
- Key terminology and product names (and anything I should never call it)
- My role in this engagement
- Style or voice rules for artifacts written on their behalf
- Confidential information handling rules

Save the result as claude-context/portco-<name>.md and update INDEX.md.

Keeping it current

The system is only as good as it stays current. Two standing rules:

  • After any session where you correct Claude's behavior — ask it to update operating-guide.md so the correction sticks next session.
  • After adding or editing any file in claude-context/ — ask Claude to update the Contents table in INDEX.md so the map stays accurate.
